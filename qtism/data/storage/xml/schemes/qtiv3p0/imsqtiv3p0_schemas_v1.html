Question & Test Interoperability 3.0

Final Specification (2022-05-01) - XML Schema and Vocabulary Documents Version 1

This is the set of Final Release XSDs developed by IMS Global to support validation of instances that conform to the IMS Question and Test Interoperability (QTI) v3.0 specification. The namespace and schema location details for the QTIv3.0 XSDs are provided in the tables below.

  1. Packaging-Related XSDs - the construction of the package manifest for the zipped QTI package;
  2. QTI-Specific XSDs - definition of the accessible QTI instances, results reports and usage data statistics;
  3. Metadata-Related XSDs - the set of metadata supported in the manifest;
  4. Access for All Personal Needs & Preferences XSDs - XSDs required for the validation of the QTI AfA PNP instances;
  5. Other Required XSDs - other XSDs required for the full validation;
  6. VDEX-based Vocabularies - the set of VDEX-based control vocabularies.

The key for the following tables is:

  • "Namespace" - the namespace for the XSD (must be supplied in any XML instance referring to the corresponding XSD);
  • "XSD File Name" - the filename for the XSD;
  • "Schema Location" - the schema location for the XSD (must be supplied in any XML instance referring to the corresponding XSD).

The following set of XSDs are used for the validation of QTIv3.0:

  • P1 - used to validate the "imsmanifest.xml" file that must be present in the zipped package;
  • P2 - used to validate the CP 1.2 manifest extension features used by QTI;
  • Q1 - used to validate a QTI instance;
  • Q2 - used to validate a QTI Item-only instance;
  • Q3 - used to validate a QTI Test-only instance;
  • Q4 - used to validate a QTI Section-only instance;
  • Q5 - used to validate a QTI Stimulus-only instance;
  • Q6 - used to validate a QTI Outcome Declaration-only instance;
  • Q7 - used to validate a QTI Response Processing-only instance;
  • Q8 - used to validate the reporting of QTI results;
  • Q9 - used to validate the usage data statistics for presented QTI items;
  • M1 - used to validate the learning standards annotations, at the manifest and resource level, defined using the Curriculum Standards Metadata (CSM);
  • M2 - used to validate the metadata, in the "imsmanifest.xml" file, defined using IEEE LOM;
  • M3 - used to validate the QTI metadata that is associated with a QTI-based resource;
  • M4 - used to validate alternative resource metadata based upon the Access for All Digital Resource Description (AfA DRD);
  • A1 - used to validate the Access for All Personal Needs & Preferences instances;
  • X1 - used to validate the xml-specific attributes;
  • X2 - used to validate the XInclude features;
  • X3 - used to validate the MathML 3 features;
  • X4 - used to validate the SSMLv1.1 features;
  • X5 - used to validate the VDEX-based vocabularies;
  • V1 - glossary of terms for Usage Data Distractor Statistics;
  • V2 - glossary of terms for Usage Data Item Statistics.
